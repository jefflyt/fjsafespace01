Section 3: Operations View Enhancements

Sparkline charts and report generation improvements for Phase 1

Sparkline Charts in Findings Tab

Each metric row in the Findings table gets a mini time series chart showing values across the upload period:

Findings Table — Per-Metric Rows
Metric Value Band Trend Standard Action
CO2 950 ppm WATCH ──╱── (sparkline) SS 554 §5.2.1 Increase ventilation
PM2.5 8 μg/m³ GOOD ─── (flat sparkline) WHO AQG Ch.6 No action required

Implementation: Recharts LineChart (already in frontend dependencies) or lightweight SVG sparkline component. Data comes from readings grouped by upload_id + metric_name.

Report Generation Flow

Unchanged from current implementation:

  1. Findings → QA checklist → Approver (Jay Choy) → JSON snapshot stored in DB
  2. PDF generated on-demand from snapshot via WeasyPrint
  3. Each finding's citation badge links to the specific standard (SS 554, WHO, etc.)

What Sparklines Enable

  • Quick visual: is a metric trending up, down, or stable?
  • No new backend API — readings already stored per timestamp
  • Phase 2 full monitoring view can reuse this component

Scope Note

  • Sparklines show data for ONE upload only (the one being reviewed)
  • No cross-upload historical trends (Phase 2 with uHoo live API)
