# Report Template Stylist

This skill governs the visual and structural integrity of the **PDF reports** generated by WeasyPrint.

## Workflows

### 1. Creating a New Report Template
1.  **Draft HTML Structure:** Use `Jinja2` syntax for dynamic findings in `backend/app/templates/`.
2.  **Define Layout:** Use `@page` rules in CSS for A4 margins and header/footer content.
3.  **Apply Logic:** Ensure template branches for `ASSESSMENT` and `INTERVENTION_IMPACT` report types.

### 2. Styling the Executive Brief
The "Executive Brief" is the most critical part of any FJ SafeSpace report:
- **Status:** Display `HEALTHY_WORKPLACE_CERTIFIED`, `HEALTHY_SPACE_VERIFIED`, etc.
- **Top 3 Findings:** Highlight metrics with the highest `alert_priority`.
- **Sign-off:** Include the reviewer name and date (QA-G8 requirement).

## Governance Guardrails
- **Citations:** Every parameter block must include a reference to the `citation_unit_id` (TDD §4.8, QA-G5).
- **Advisory Labels:** Findings from non-`CURRENT_VERIFIED` sources must be clearly labeled as "Advisory Only" (QA-G6).
- **Reproducibility:** The template should not contain hardcoded strings that belong in the database (e.g., threshold values).

## Resources
- **CSS:** See `references/weasyprint_cheatsheet.md` for WeasyPrint-specific styles.
- **Paths:** Templates are located in `backend/app/templates/`.

## Previewing a Template
To preview a template change, use the `scripts/preview_report.py` script with sample JSON data from `assets/`.
```bash
python scripts/preview_report.py --template assessment.html --data assets/sample_finding_data.json --output preview.pdf
```
